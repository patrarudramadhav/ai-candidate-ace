# Talent Scout AI

> AI-powered candidate discovery — built for the **Deccan AI Catalyst Hackathon**

Paste a Job Description, let Gemini 2.5 Flash score a pool of 8 diverse candidates across two independent dimensions, and export a ranked shortlist as CSV — all in under 10 seconds.

---

## Demo

| Step | What happens |
|---|---|
| 1. Paste a JD | Any role, any domain — fintech, NGO, arts, engineering |
| 2. Click "Scout Candidates" | Edge function sends all 8 profiles + JD to Gemini |
| 3. View ranked table | Weighted score = Match × 0.6 + Interest × 0.4 |
| 4. Open Analysis modal | Per-candidate chat transcript, skill breakdown, justifications |
| 5. Export CSV | One-click shortlist download with all scores & reasoning |

## Scoring Logic

### Two-score model

Every candidate receives **two independent scores** (0–100), each generated by Gemini from a different perspective:

| Score | What it measures | Weight |
|---|---|---|
| **Match score** | Skill, seniority, and experience alignment with the JD | 60% |
| **Interest score** | How likely the candidate is to actually move | 40% |

**Final weighted score = `matchScore × 0.6 + interestScore × 0.4`**

Candidates are ranked descending by weighted score. The top-ranked candidate is the one who is both a strong fit *and* realistically pursuable.

---

### Match score — how it works

The model is prompted to act as an experienced technical recruiter and score conservatively. For each candidate it:

1. Extracts 4–6 **key requirements** directly from the JD (e.g. "TypeScript", "5+ yrs experience", "Fintech domain", "Team leadership").
2. Checks each requirement against the candidate's profile: `match`, `partial`, or `missing`.
3. Assigns a **weight (1–10)** to each requirement based on how decisive it was for this specific candidate.
4. Derives a 0–100 score grounded in this skill breakdown — not a generic gut-feel.
5. Writes a **2-sentence justification**: what aligns, then what is missing or weaker.

A candidate from a completely different domain (e.g. a traditional artist scored against a software JD) correctly receives a very low match score, because the prompt instructs the model to be honest rather than generous.

---

### Interest score — how it works

The interest score is **derived from a simulated outreach chat**, not scored in isolation. The model:

1. Generates a **4-message DM transcript** (agent → candidate → agent → candidate).
   - Message 1 (agent): warm intro referencing the specific role and one of the candidate's actual skills.
   - Message 2 (candidate): realistic reply — tone reflects seniority and domain (a Staff engineer at Razorpay sounds different from a junior).
   - Message 3 (agent): follow-up on availability, compensation, or interest level.
   - Message 4 (candidate): candid signal — how open (or not) they are to moving.
2. Reads its own simulated chat to **derive the interest score**. If the candidate's replies sound dismissive or settled → low score. Curious and available → high score.
3. Writes a **2-sentence justification** citing specific phrasing or signals from the chat.

This makes the interest score self-consistent and explainable: you can read the chat transcript and verify why the score is what it is.

---

### Why tool-calling instead of free-form JSON

The edge function uses **forced tool-calling** (`tool_choice: { type: "function", function: { name: "score_candidates" } }`) with a strict JSON schema. This guarantees:

- All required fields are present for every candidate.
- Numeric scores stay in the 0–100 range.
- `status` values are always one of `"match" | "partial" | "missing"`.
- The chat transcript always has exactly 4 messages alternating agent/candidate.
- No markdown fences or prose leak into the structured output.

## Running Locally

```bash
# Install dependencies
bun install

# Start dev server
bun run dev
```

You will need a `LOVABLE_API_KEY` set in your Supabase project's Edge Function secrets, and the `scout-candidates` function deployed:

```bash
supabase functions deploy scout-candidates
```

## Error Handling

| Condition | Response |
|---|---|
| JD under 20 characters | Client-side toast, request blocked |
| More than 25 candidates | Edge function rejects with 400 |
| Rate limit (429) | Toast: "Please wait a moment and try again" |
| AI credits exhausted (402) | Toast: "Add credits in Settings → Workspace → Usage" |
| AI returns no tool call | Edge function returns 500, table not cleared |
